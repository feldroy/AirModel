# AirModel 0.4.0: Auto-migration for new model fields

Add a field to your model, call `create_tables()`, and the column appears in PostgreSQL. No manual `ALTER TABLE`, no migration tool for the common case of adding fields to an existing model.

```sh
uv add --upgrade AirModel
```

### What's new

- **`create_tables()` migrates existing tables.** After running `CREATE TABLE IF NOT EXISTS`, AirModel checks `information_schema.columns` and runs `ALTER TABLE ADD COLUMN` for any model fields not yet in the database. Non-destructive: never drops columns, never changes types. New columns are added without `NOT NULL` so existing rows stay intact, while Pydantic still enforces requirements at the app layer.

### What's better
