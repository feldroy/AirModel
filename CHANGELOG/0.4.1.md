# AirModel 0.4.1: CI green across all checks

The type checker passes cleanly on the full test suite, including the migration pool mock introduced in 0.4.0.

```sh
uv add --upgrade AirModel
```

### What's fixed

- **Type checker no longer flags `MigrationPool.fetch`.** The mock pool's `fetch` method received `args` typed as `tuple[object, ...]`, making the dict key lookup ambiguous to the type checker. The key is now explicitly converted to `str`.
