# AirModel 0.1.1: Async ORM that turns Pydantic models into PostgreSQL tables

Your Pydantic models are your database schema. Define them with type annotations, and AirModel gives you async `create`, `get`, `filter`, `all`, `count`, `save`, and `delete` against PostgreSQL, with Django-style lookups like `sparkle_rating__gte=8` and `location__icontains="falls"`.

```bash
uv add AirModel
```

Extracted from [Air PR #1101](https://github.com/feldroy/air/pull/1101) into its own package so non-Air projects can use it too.

### What's in the box

- **Full async CRUD** with `create`, `get`, `filter`, `all`, `count` as class methods, plus `save` and `delete` on instances. Every query method returns typed Pydantic model instances.
- **Django-style lookups** in any query: `__gt`, `__gte`, `__lt`, `__lte`, `__contains`, `__icontains`, `__in`, `__isnull`. Combine them freely: `filter(sparkle_rating__gte=8, confirmed=True)`.
- **Bulk operations** that minimize round trips: `bulk_create` inserts many rows in a single query, `bulk_update` and `bulk_delete` return affected row counts.
- **Transactions** via `async with db.transaction()` that route all CRUD through a single connection, with automatic commit on success and rollback on exception.
- **Partial updates** with `save(update_fields=["name"])` to SET only the columns you changed.
- **Automatic table creation** with `db.create_tables()`, which runs `CREATE TABLE IF NOT EXISTS` for every registered model.
- **Zero-config with Air**: set `DATABASE_URL` and Air auto-discovers AirModel, opens a connection pool on startup, and exposes it as `app.db`.

Built on [asyncpg](https://github.com/MagicStack/asyncpg) and [Pydantic v2](https://docs.pydantic.dev/). Requires Python 3.13+.
