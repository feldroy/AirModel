## AirModel 0.1.2: Field metadata moves to AirField

AirModel's `Field` function lives in the [AirField](https://github.com/feldroy/AirField) package as `AirField`. Import it directly instead of through AirModel.

```sh
uv add AirModel>=0.1.2
```

### What's changed

- **`from airmodel import Field` becomes `from airfield import AirField`**, and `Field(...)` calls become `AirField(...)`. AirModel re-exports `AirDB`, `AirModel`, and `MultipleObjectsReturned` but no longer re-exports the field function.
